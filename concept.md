## 1. Проблема

Подготовка школьников к ОГЭ и ЕГЭ по информатике неэффективна из-за:

- **Отсутствия мгновенной обратной связи** - ученик не понимает, где и почему ошибся в коде без помощи учителя
- **Невозможности потренироваться в решении конкретных задач из КИМ** - существующие ресурсы предлагают разрозненные задачи без системы
- **Страха и непонимания программирования** - абстрактные концепции и алгоритмы не подкреплены визуализацией
- **Для учителя** - сложно уделить время каждому ученику для разбора ошибок в коде

## 2. Цель

Разработать веб-платформу на JavaScript-стеке для помощи ученикам в подготовке решения задач ЕГЭ по информатике со встроенным компилятором с функцией получения мгновенной проверки через безопасную среду выполнения на сервере.

## 4. Архитектура

### 4.1. Бизнес-архитектура (Анализ)

- **Ценностное предложение**: Мгновенная проверка решений на Python для задач ОГЭ/ЕГЭ по информатике
- **Каналы продвижения**: Сообщества учителей информатики, SEO, партнерства с репетиторами
- **Ключевые метрики**: DAU/MAU, количество решенных задач, конверсия в подписку
- **Анализ рисков**: Сложность безопасного выполнения Python-кода компенсируется использованием контейнеризации

### 4.2. Программная архитектура (JavaScript Backend + Python Execution)

**Технологический стек**:

**Frontend (JavaScript/TypeScript)**:
- Next.js 14+ (App Router), TypeScript, Tailwind CSS
- Monaco Editor с подсветкой синтаксиса Python
- React Hook Form для управления формами

**Backend (Node.js)**:
- Express.js или Nest.js для основного API
- JWT аутентификация
- MongoDB с Mongoose ODM
- Очередь задач (Bull.js) для асинхронной проверки решений

**Python Execution Service (Python)**:
- FastAPI для lightweight API
- Docker для изоляции выполнения кода
- Изолированные контейнеры для каждого запуска
- Ограничения по времени и памяти

**Инфраструктура**:
- Vercel (frontend)
- Railway/Render (Node.js backend)
- Отдельный сервер/VPS для Python execution service

### 4.3. Аппаратная архитектура

- **Фронтенд**: Vercel (бесплатный хостинг для Next.js)
- **Бэкенд API**: Railway/Render (бесплатные тарифы)
- **База данных**: MongoDB Atlas (бесплатный кластер 512MB)
- **Python Execution**: DigitalOcean Droplet/Hetzner VPS (~500 руб./мес)

## 5. Взаимодействие

### Пользовательские сценарии

1. **Ученик**:
   - Регистрируется через email
   - Выбирает задачу по номеру КИМ или теме
   - Пишет решение на Python в редакторе с подсветкой синтаксиса
   - Нажимает «Проверить» → код отправляется на сервер
   - Python Execution Service запускает код в Docker-контейнере
   - Ученик видит результаты выполнения и прохождение тестов

2. **Разработчик**:
   - Использует TypeScript для основной разработки
   - Управляет Python-сервисом через REST API
   - Добавляет задачи через админку на Next.js

## 6. Безопасность
   **Безопасность выполнения Python-кода**
   
   - *Docker изоляция*: Каждый запуск в чистом контейнере

   - *Ограничения ресурсов*:

      - 5 секунд на выполнение

      - 128MB RAM

      - Нет доступа к сети

   - *Безопасный образ Python*: Минимальный образ без опасных пакетов

   - *Проверка кода*: Статический анализ на запрещенные операции

   **Безопасность приложения**

   - *Next.js Security*: Встроенная защита от XSS, CSRF

   - *API Security*: Валидация через Zod, rate limiting

   - *Authentication*: NextAuth.js с провайдерами OAuth

   - *Data Validation*: TypeScript + runtime валидация
